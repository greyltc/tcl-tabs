Description

Tabs bases on jobs that can be run either standalone or as part of Tabsfile. Each job is created as a snit object and has its own jobmanager. A job is run using tabs::runjob command, which creates a manager as well as actual job to perform.

Manager stores common options such as -label and performs checks

This mechanism should only be used by advanced users and is only documented for reference.

COMMANDS

::tabs::runjob type ?options?

Creates a job manager, job object and runs specified job. All options are passed to object as configuration options. Specified job type needs to accept all of specified options.

::tabs::jobs::<type> ?options?

Creates a new job object. This is used internally by tabs::runjob command and is documented mainly for developers creating new job types. All methods specified in this documentation have to be implemented by all jobs.

job execute

Performs specified job.

job inputs

If implemented, this method should return a list of all input files and directories. This is used to determine whether a job should be re-run.

See Checking whether job should be run for more details on determining is a job should be run.

job outputs

If implemented, this method should return a list of all output files and directories. This is used to determine whether a job should be re-run.

See Checking whether job should be run for more details on determining is a job should be run.

job uptodate

If implemented, this method should return whether result of the job is up to date or if it should be re-run.

See Checking whether job should be run for more details on determining is a job should be run.

job label

If implemented, should return the default label for a task if one was not specified using -label option. For example it is implemented as follows for tabs::jobs::wrap type:

method label {} {
    return "wrap [$self cget -output]"
}

Options provided by job manager

The following options are handled by job manager and are accepted by all job types.

When writing a custom job, you should access these options in the following way:

if {[[$self cget -manager] cget -debug]} {
    # perform additional debug actions
}

  • -always - if set to true, job will be used regardless if it is up to date or not defaults to false; see Checking whether job should be run for more details on determining is a job should be run

  • -debug - if specified, additional information can be printed by a specified job defaults to false

  • -skiptags - when built using Tabsfile, specifies list of tags in which cases a task should be skipped; defaults to empty list

  • -internal - specifies that a job is used as a delegate of another job - for example a tabs::jobs::wrap job creates an internal tabs::jobs::copy job to perform copying

  • -label - label to use for output when building using Tabsfile; defaults to empty string, in which case task label is generated using job type's label method
